# 发票识别、分类与归档

## 核心原则

- 先读取 PDF 内嵌文本；仅在文本缺失或不可用时执行 OCR。
- 涉及发票上传到外部 OCR 服务时，先取得用户许可。优先使用本地 PDF 解析或本地视觉模型。
- 以金额、日期、发票号码和路线交叉验证，禁止仅凭文件排列顺序配对。
- 重命名前先输出或检查 dry-run 结果；存在金额重复、缺少配对或目标文件冲突时停止，不猜测。
- 修改文件前保留同目录边界，禁止移动到任务目录之外。
- 按行程归档时必须执行行程完整性检查；交通未闭环或住宿未逐晚覆盖时明确提醒用户，不得静默视为完整行程。

## 简化重命名规则

用户要求按时间、类型、金额分类或未指定详细命名格式时，统一使用：

```text
YYYY.MM.DD_类型_金额元.扩展名
```

跨多日凭证使用：

```text
YYYY.MM.DD至YYYY.MM.DD_类型_金额元.扩展名
```

- 日期优先使用实际发生日期；无法取得时才使用开票日期。
- 滴滴发票与行程单使用相同的实际行程日期区间。
- 类型使用简短、明确的名称，如 `油票`、`快递发票`、`快递订单`、`滴滴发票`、`滴滴行程单`。
- 默认不在文件名中加入公司名称、城市、发票号码、订单号码或用途描述。
- 仅当目标文件名冲突时，在金额后追加最少必要的区分信息；不得覆盖已有文件。
- 重命名前先执行或人工检查 dry-run，完成后按名称排序并核对文件数量。

## 滴滴发票与行程单成对重命名

当目录同时包含 `滴滴电子发票*.pdf` 和 `滴滴出行行程报销单*.pdf` 时，运行：

```powershell
python scripts/rename_didi_pairs.py "待处理目录" --dry-run
python scripts/rename_didi_pairs.py "待处理目录"
```

脚本通过价税合计金额与行程单合计金额配对，并从行程单提取时间区间。最终命名格式：

```text
YYYY.MM.DD至YYYY.MM.DD_滴滴发票_金额元.pdf
YYYY.MM.DD至YYYY.MM.DD_滴滴行程单_金额元.pdf
```

若脚本生成了包含城市的旧格式，按“简化重命名规则”移除城市信息。执行后验证：

- 发票数与行程单数相等。
- 每组基础名称一致，仅“滴滴发票/滴滴行程单”不同。
- 文件名金额与两份 PDF 的金额一致。
- 原始通用名称文件数为 0。

## Excel 模板

模板资产：

```text
assets/材料、测试加工、差旅费及其他费用明细表.xlsx
```

始终先复制模板到输出目录，再填写副本；禁止直接修改技能内的资产文件。处理工作簿时使用电子表格技能并保留现有格式。

工作表结构：

- `材料测试加工等费用`：数据从第 5 行开始，B-F 列依次为序号、费用类别、金额、用途、备注。
- `差旅等费用`：数据从第 6 行开始，B-G 列依次为序号、差旅日期、出差人员、差旅金额、其他金额、说明。
- `其他日常费用`：数据从第 9 行开始，C-G 列依次为序号、费用类别、金额、用途、其他说明。

填写差旅表时：

- 日期区间使用 `YYYY.MM.DD\n-YYYY.MM.DD`，保持换行显示。
- 差旅金额包含火车票、飞机票和住宿。
- 其他金额包含滴滴/的士、餐费、租车、加油和退票费。
- 金额保持数值或可审计的加法表达式，发票号码保持文本格式。
- 填写后检查公式错误并渲染所有工作表做视觉核验。

## 行程完整性检查

按行程分类或填写差旅表时，逐个行程检查交通和住宿，并在输出汇总中增加“完整性状态”和“待复核事项”：

- **交通闭环**：以实际乘车、乘机凭证确认从出发基地离开并返回基地。缺少去程、缺少回程、路线中断或相邻交通段城市无法衔接时，标记 `未闭环` 并提醒用户。
- 退票费、保险、订票服务费和开票记录不能单独证明实际出行，不得用于补齐交通闭环。
- **住宿逐晚覆盖**：对跨日行程检查从出发日到返程日前一晚的每个住宿夜。每一晚都必须有住宿凭证覆盖；当天往返行程不要求住宿。
- 住宿覆盖优先使用入住日和离店日区间判断。只有开票日期、没有入住/离店日期的住宿发票不能直接证明覆盖，应标记待复核。
- 夜间火车、夜间航班等可能替代住宿时，仅在凭证时间明确且能够覆盖该住宿夜时列为“夜间交通（待人工确认）”，不得自动视为住宿齐全。
- 任一项不满足时，保留归档结果，但文件夹名称追加 `_未闭环`，并在汇总表或处理报告中逐项列出，例如 `缺少回程交通`、`路线断点：西安→上海`、`缺少住宿：2026-06-16晚`。
- 最终回复必须主动报告未闭环行程数量、名称及缺失项；没有异常时明确说明交通闭环且住宿逐晚齐全。

## 通用发票工作流

1. 递归清点 PDF、图片、已有 OCR JSON 和模板文件。
2. 提取内嵌文本；必要时按页渲染并执行 OCR。
3. 提取发票号码、开票日期、实际发生日期、金额、销售方、出发地和目的地。
4. 按退票费、火车票、飞机票、住宿、租车、加油、餐饮、打车、材料费、其他分类。
5. 用实际发生时间排序，并用实际交通凭证识别路线和往返闭环。
6. 按住宿夜逐日检查住宿凭证覆盖，生成未闭环和待复核提醒。
7. 生成汇总表、归档副本或填写模板，最后核对文件数、金额合计和行程完整性状态。

需要解析规则、城市映射、行程闭环或模板字段实现细节时，读取
[`references/detailed-invoice-workflow.md`](references/detailed-invoice-workflow.md)。
